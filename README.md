# CSV Median Calculator

Консольное приложение для инкрементального расчёта медианы и других статистических метрик (среднее, стандартное отклонение, квантили p90/p95/p99) из потоковых биржевых данных CSV-файлов.

## Возможности

- Потоковая обработка CSV-файлов любого размера (без загрузки в память)
- Инкрементальный расчёт статистики по мере поступления данных
- Поддержка типов файлов: `*.csv`
- Гибкая фильтрация файлов по маскам
- Точность вывода: 8 знаков после запятой
- Логирование через `spdlog`
- Обработка ошибок без аварийного завершения
- Модульные тесты (Catch2)

## Требования

- CMake 3.23+
- Компилятор с поддержкой C++23 (GCC 14+, Clang 17+)
- Boost (filesystem, program_options)
- spdlog
- toml++ (автоматически скачивается)
- Catch2 (для тестов, автоматически скачивается)

## Сборка

```bash
# Клонирование репозитория
git clone <repository-url>
cd csv_median_calculator

# Создание директории для сборки
mkdir build && cd build

# Конфигурация CMake
cmake ..

# Сборка
make

# Запуск программы
./csv_median_calculator [--config <path>]
```

## Конфигурация

Приложение использует TOML-файл конфигурации. Пример `config.toml`:

```toml
[main]
    # Путь к директории с входными CSV-файлами (обязательно)
    input = "examples/input"
    
    # Путь к выходной директории (опционально, по умолчанию ./output)
    output = "examples/output"
    
    # Маски для фильтрации файлов (опционально)
    # Если не указано - обрабатываются все .csv файлы
    filename_mask = ["level", "trade"]
```

### Параметры командной строки

```bash
./csv_median_calculator --config <path>   # альтернативный вариант
```

Если аргументы не указаны, приложение ищет `config.toml` в директории проекта

## Формат входных файлов

### level.csv

```csv
receive_ts;exchange_ts;price;quantity;side;rebuild
1716810808593627;1716810808574000;68480.10000000;10.10900000;bid;1
```

### trade.csv

```csv
receive_ts;exchange_ts;price;quantity;side
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid
```

## Выходной файл

Результат сохраняется в `stat_result.csv` с колонками:

- `receive_ts` – временная метка события, изменившего статистику
- `median` – текущая медиана
- `mean` – среднее арифметическое
- `stddev` – стандартное отклонение
- `p90` – 90-й процентиль
- `p95` – 95-й процентиль
- `p99` – 99-й процентиль

Все значения выводятся с точностью 8 знаков после запятой.

## Архитектура

- **CSVReader** – основной класс для чтения и обработки CSV-файлови
- **FileState** – состояние конкретного файла
- **StatsCalculator** – инкрементальный расчёт статистики (гибрид: точный до порога, затем Boost.Accumulators)
- **Config** – загрузка и валидация конфигурации TOML

## Принцип работы

1. Поиск всех CSV-файлов в указанной директории с учётом масок
2. Инициализация каждого файла (чтение заголовка и первой строки)
3. Построение min-кучи из первых событий для хронологической сортировки
4. Цикл обработки:
   - Извлечение самого раннего события из кучи
   - Добавление цены в статистику
   - Если медиана изменилась – запись в выходной файл
   - Чтение следующей строки из того же файла и добавление в кучу
5. Повтор до опустошения кучи
